Configuration Folder

  • Location
  • Default contents
  • Additional contents

The configuration folder stores the majority of the OmegaT parameters and preferences for the user.

You can always access it directly from Access Configuration Folder in the Options menu.

Note

If anything happens to your OmegaT installation and you can't seem to fix it, close OmegaT, make a backup of the contents of the configuration folder, delete the original configuration folder, and restart OmegaT. You can then copy your backed up configuration files to restore your original preferences.

If you are recovering from a corrupt installation, you should copy the backed up files one at a time to make it easier to isolate the problem if it persists.

Location

The location of the configuration folder varies by system (the ~ character represents your home directory):

Linux

~/.omegat

macOS

~/Library/Preferences/OmegaT

Windows

~\AppData\Roaming\OmegaT

Default contents

omegat.prefs

This file includes a number of important user preferences.

Some preference do not have an equivalent in the user interface. They must be modified manually.

Example 16. Do not automatically display the Source Files list

To keep the Source Files list window to automatically open when a project is loaded, find <project_files_show_on_load> and replace true to false:

<project_files_show_on_load>false</project_files_show_on_load>


Note

Only this preference currently requires manual modification.

uiLayout.xml

This file describes the overall OmegaT layout.

logs/

This folder contains a number of log files. The most current is OmegaT.log.

These files record various internal state and program event messages generated while OmegaT is running. If OmegaT behaves erratically, it is important to include this file, or the relevant part thereof, if you submit a bug report.

The contents of the file can also be viewed using the Log... option in the Help menu.

script/

If the applicable functions are used, this folder can contain up to three text files:

selection.txt

This file stores the currently selected text when the Export Selection C + C + S command in the Edit menu is used. The text in the file is replaced each time this function is called.

source.txt

This file contains the original text from of the current segment when Preferences – Export the segment to text files is active. The text in the file is replaced each time a new segment is entered.

target.txt

This file contains the translated text from the current segment when Preferences – Export the segment to text files is active. The text in the file is replaced each time a new segment is entered.

Those three files provide as a simple way to access some OmegaT content and process it with local programs such as shell scripts.

Additional contents

EditorShortcuts.properties

This parameter file contains customized editor shortcuts. See Customization for details.

MainMenuShortcuts.properties

This parameter file contains customized user interface shortcuts. See Customization for details.

filters.xml

This parameter file contains customized file filters. See Preferences – Global File Filters for details.

finder.xml

This parameter file contains customized external search parameters. See Preferences – Global External Searches for details.

omegat.autotext

This parameter file contains customized autotext parameters. See Auto-Completion for details.

repositories.properties

This file contains the login information for your team project repositories.

Warning

The file contents are not encrypted.

See Set up a team project for details.

segmentation.conf

This parameter file contains customized segmentation parameters. See Preferences – Global Segmentation Rules for details.

plugins/

This folder provides an alternative location for manually installed OmegaT extension plugins. See Preferences – Plugins for details.

spelling/

This folder contains your spelling dictionaries. See Install new dictionary... for details.

Note

You can specify a configuration folder other than the default when you start OmegaT from the command line. See Command line launch for details.

Preferences are stored in the default configuration folder. If you specify a different configuration folder, all modifications made in the Preferences dialog will be stored in the newly specified configuration folder.
