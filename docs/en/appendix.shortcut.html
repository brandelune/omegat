Description

OmegaT generally does not use buttons to give you access to its functions. The functions are available from the menus and most of them have been assigned default shortcuts.

You will only need a few minutes of working with OmegaT to remember the most frequent shortcuts, and you will always be able to learn new shortcuts from the menus.

Almost all of OmegaT’s shortcuts are customizable. See Customization for details.

OmegaT runs on any platform that runs a Java Runtime Environment (Windows, macOS, Linux being the most mainstream). The modifier keys that compose the shortcuts vary slightly depending on the platform. To make reading easier we have adopted the following presentation for modifier keys:

Table F.1. Modifier key description
Linux/Windows Key description macOS
Shift S shift or ⇧
Control C command or ⌘
Alt A alt / option or ⌥
  Ctrl control or ⌃


We can thus use only the Key description codes above to avoid repeating multiple notations.

Example F.1. How we avoid repeating multiple notations:

On Windows and Linux: Control + Shift + N

On macOS: Shift + Command + N

In this manual: C + S + N


Customization

OmegaT assigns shortcuts to most functions available in the menus and to a number of functions in the Editor window. Most of these functions can be assigned new shortcuts.

You can change or add a shortcuts by putting a shortcut definition file in your OmegaT configuration folder. See Configuration Folder for details.

There are 2 shortcut definition files.

MainMenuShortcuts.properties

The shortcut definition file for the menus and a few other items.

EditorShortcuts.properties

The shortcut definition file for the editor.

Once a shortcut definition file is modified, OmegaT must be relaunched to take the new shortcuts into account.

Note

You can copy the default OmegaT shortcut files from the OmegaT development site on Sourceforge to your configuration folder and modify them to suit your needs:

Windows and Linux default shortcuts

  • MainMenuShortcuts.properties

  • EditorShortcuts.properties

macOS default shortcuts

  • MainMenuShortcuts.mac.properties

  • EditorShortcuts.mac.properties

The macOS files must be renamed MainMenuShortcuts.properties and EditorShortcuts.properties to be taken into account by OmegaT.

Syntax

Empty lines are accepted and comment lines should start with //. Anything after the // will be ignored.

The shortcut definition syntax is the following:

function code=shortcut

function code is a code taken from the tables below.

shortcut is a combination of pressed keys specified by the user.

shortcut must be of the following form:

  • 0 or more modifier

  • followed by 0 or 1 event

  • followed by 1 key

where:

  • modifier can be: shift , control , ctrl , meta , alt , altGraph

    Note

    meta is used to describe command on macOS.

  • event can be: typed , pressed , released

  • and key can be any key available on your keyboard.

Example F.2. Open Project shortcut modification

The default shortcut for Open Project is defined this way:

projectOpenMenuItem=ctrl O

or, on macOS

projectOpenMenuItem=meta O.

If you want to add Shift to the definition, modify the MainMenuShortcuts.properties as follows:

projectOpenMenuItem=shift ctrl O.

or, on macOS:

projectOpenMenuItem=shift meta O.


After the modifications, save the file and relaunch OmegaT. Your new shortcuts should now appear next to the menu items you have modified. If they do not conflict with system shortcuts, they should be available from within OmegaT.
