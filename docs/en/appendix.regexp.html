This appendix is intended for users interested in exploring a powerful way to boost their productivity. Although seen as daunting and complex, even the simplest regular expressions (often abbreviated regex or regexp ) are extremely useful, not only in OmegaT, but in many other applications you might use on a day-to-day basis.

Only the fundamentals most useful to translators are covered. The References section at the end of this appendix provides a few starting points to explore advanced or complex uses beyond the scope of this manual. If you need help for a specific case, you can also ask questions in the various support channels.

Regular expressions generally use a combination of letters, digits, and symbols (collectively known as characters ) to define an expression that represents a specific text pattern.

Here are a few examples.

[0-9]

Any single digit from 0 to 9.

\w+

Represents one or more "word characters", namely the letters of the alphabet, digits, as well as the hyphen and undescore symbols.

\h?

Represents zero or one horizontal whitespace character (this includes regular and non-breaking spaces as well as tabs, but not line break characterss, which belong to the "vertical whitespace" category (\v).)

Many OmegaT functions rely on regular expressions or make them available as an option:

Searches

OmegaT searches include a Regular expressions option that allows you to make extremely powerful searches across your files.

The same option in the Text Replace dialog allows you to apply regular expressions to both the search and replaced text.

Custom tags

Custom tags are tags defined with regular expressions that are handled exactly like native OmegaT tags. See Regular expressions for custom tags for details.

Use the | (OR) character to separate individual tag definitions.

Flagged fragments

Regular expressions for fragments that should be removed from translation allow you to define strings that OmegaT will mark in red by default, and treat as extraneous tags for validation purposes.

Use the | (OR) character to separate individual fragment definitions.

Segmentation

Segmentation rules and language patterns are defined with regular expressions. You can modify them freely to improve the segmentation of a document or add additional general rules. See Segmentation Setup for details.

Segmentation or exception rules define the position in a segment where a split will, or will not, be made. Two regular expressions are required to define that position: a "before" expression to define the text pattern ahead of where the rule should apply, and an "after" expression to define the text pattern following that position.

A language pattern that matches the source language of the project will apply to that project.

Basic principles

Regular expressions are used to find text (including characters, such as spaces, tabs, or line breaks that are not visible on the screen or when printed out. Any given expression either matches , or does not match a word, phrase, or other sequence of text.

Each and every character in the expression is relevant when determining a match.

A number of characters or combinations of characters have a special meaning in a regular expression.

Warning

Text decorations such as bold , italics , or other stylistic effects cannot be matched using regular expressions.

There are four principles to keep in mind.

Most characters simply match themselves

The majority of characters in a regular expression simply look for themselves in the text sequence.

For example, the seven letters spelling out the word " example " simply tell the search function to match exactly those letters, in that order. Simply put, the search just looks for the word " example ".

Letters of the alphabet preceded by a blackslash (\) take on a special meaning

Unlike a letter on its own, which simply represents itself as noted above, a letter preceded by a \ has a special function in a regular expression.

For example, r is just a normal character but preceding it with \ to make it \r turns it into a special combination that matches a carriage return character . Similarly, \R matches any line break character ).

Note

In the version of Java used to build OmegaT, only the letters i j l m o , and y , in both lower and upper case, have no hidden special meaning. This manual does not describe the special meanings of all the other characters. Consult the sites in the References section below for information on combinations not covered here.

Twelve characters have a special meaning by default

To match themselves, the special effect must be disabled, by a special effect character.

For example, . has the default special effect of matching any one character .

To find a normal period, it is necessary to disable the special effect with \, in which case it becomes \. and matches only a period.

In the version of Java used to build OmegaT, those twelve characters are the backslash \, the caret ^, the dollar sign $, the period or dot ., the vertical bar or pipe symbol |, the question mark ?, the asterisk or star *, the plus sign +, parentheses (), brackets [], and curly braces {}.

The BACKSLASH: \

This character either cancels or activates the special meaning of the following character.

Example B.1. Canceling a special meaning

\. = cancels the period's special meaning of matching any one character, and turns it into a plain period.


Example B.2. Activating a special meaning

\b = no longer just the letter "b", but now rather a marker for word boundaries.


The CARET: ^

When it is the first character in the expression, the caret character matches the beginning of a line.

Example B.3. Matching the beginning of a line.

^A = an uppercase A at the beginning of a line.


When it is the first character of an expression enclosed in square brackets, see below), the caret negates the characters that follow.

Example B.4. Negating characters

[^abc] = any characters other than "a", "b", or "c".


$

As the last character of an expression, the DOLLAR sign matches the end of a line.

Example B.5. 
^$ = an empty line (there is nothing between the beginning and the end of the line).


.

The PERIOD matches any one character.

Example B.6. 
^...$ = A three character "string" between the begining and the end of a line.


|

The VERTICAL BAR allows an alternative between two possibilities (OR).

Example B.7. 
^An |^The  = the string "An " OR the string "The " at the beginning of a line.


?

The QUESTION MARK quantifies the expression that preceeds it as 0 or 1 instances of that expression.

Example B.8. 
 an?  = a space, followed by a, followed by 0 or 1 n, followed by a space.


*

The ASTERISK quantifies the expressions that preceeds it as 0 or MORE instances of that expression.

Example B.9. 
\w+ = 0 or more WORD characters.


+

The PLUS quantifies the expression that preceeds it as 1 or MORE instances of that expression.

Example B.10. 
[\h\v]+ = 1 or MORE instances of either an horizontal white space or a vertical white space.


(

The OPENING PARENTHESIS opens a group that must be closed by a CLOSING PARENTHESIS. The group contents is memorized and can be called by a special effect number.

Example B.11. 
(\b\w+\b)\h\1\b = a group of 1 or more WORD characters that begin at a word boundary (\b) and ends at a word boundary (the second \b). That group defines a word, that is followed by an horizontal white space, followed by the contents of the group (the word), before a word boundary, which means, a word followed by itself.


)

The CLOSING PARENTHESIS closes a group and cannot be used on its own without triggering an error.

[

The OPENING SQUARE BRACKET opens a range of characters which must be closed by a CLOSING SQUARE BRACKET.

Example B.12. 
li[cs]en[cs]e = all the instances of a misspelled license.


{

The OPENING CURLY BRACE opens a quantifier that must be closed by a CLOSING CURLY BRACE. The quantifier can be an exact number, a minimum, a maximum or a range.

Example B.13. 
\d{4}/\d{1,} = 4 digits followed by a / followed by at least 1digit.


The character \ is a very special character

As you have seen above, the character \ has a special effect by default and that special effect is to either trigger or disable special effects in other characters. When it preceeds a character that has no special effect (neither natural nor hidden), it has no effect.

\ can be disabled by another \ , in which case it becomes \\ and matches what is called a backslash .

Table B.2. Classes and case

Note

In OmegaT a search is by default case-insensitive: it will match upper-case characters as well as lower-case characters. You can enable case sensitivity as an option for the whole search, but if you want finer settings you can mix (?i) before case-insensitive parts and (?-i) before case sensitive parts.

Expression Match
\w A word character, generaly defined as [A-Za-z0-9_] but in Java as [\p{Alpha}\p{gc=Mn}\p{gc=Me}\p{gc=Mc}\p{Digit}\p{gc=Pc}\p{IsJoin_Control}].

\W is NOT a word character ([^\w]).
\d A digit, generaly defined as [0-9].

\D is NOT a digit.
\s A white space that includes spaces, tabs, line breaks, etc.

\S is NOT a white-space character ([^\s]).
\h and \v Horizontal and vertical white space (generally prefered to \s).

\H is NOT an horizontal white space and \V is NOT a vertical white space.
[abc] a, b, or c

A simple class is described by any number characters between [ and ]
[C-X] A character among the "C through X" letters set

A range is described by a character that starts a series, followed by a hyphen, followed by a character that ends the series. Any number of ranges can be defined this way: [a-zA-Z0-9]. A hyphen that is outside a series is a hyphen: [a-z-].
[^\n\r\t] Any character except a newline, a carriage-return, or a tabulation

The CARET excludes what follows it.
(?i)o(?-i)megat A case insensitive "o" followed by a case sensitive "megat".

The HYPHEN cancels the "case insensitive" mode.

Table B.4. Locations

Locations (boundaries) are not part of the match. They just specify where the match should be.

Expression Match
^ The beginning of the line
$ The end of the line
\b A word boundary
\B NOT a word boundary
(?=u) Before the character "u"
(?!u) NOT before the character "u"

Table B.5. Examples of regular expressions that use the expressions above
Regular expression Finds the following:

[\.,]\s*[\.,]+ comma or a period, followed by spaces and yet another comma or period
